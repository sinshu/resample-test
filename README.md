# NumFlat resample comparison

NumFlat 1.3.2（旧版）と1.3.4（新版）の `Resample` の結果と実行時間を比較するプロジェクトです。

## 確認内容

固定シードで生成した60秒間の乱数信号に対し、フィルター次数50で次の2ケースを確認します。

| ケース | 入力 | 呼び出し | 出力 |
| --- | ---: | --- | ---: |
| 3倍ダウンサンプリング | 48 kHz / 2,880,000 samples | `Resample(16000, 48000, 50)` | 16 kHz / 960,000 samples |
| 3倍アップサンプリング | 16 kHz / 960,000 samples | `Resample(48000, 16000, 50)` | 48 kHz / 2,880,000 samples |

`p` と `q` には約分した比率ではなく、出力側・入力側のサンプリング周波数をそのまま指定しています。比率は従来と同じため、理論上のリサンプリング動作は変わりません。

新旧版の出力を倍精度浮動小数点数のまま比較し、最大絶対誤差が `1e-9` 以下なら一致と判定します。WAVへの変換は行わないため、量子化によって差が隠れることはありません。

実行時間はJITコンパイルのウォームアップ後に各ケースを5回実行した中央値です。各測定前にGCを実行しています。

## 実測結果

2026-07-18にWindows 10（10.0.26200）、.NET SDK 10.0.302、Release構成で測定しました。

| ケース | NumFlat 1.3.2 | NumFlat 1.3.4 | 新版の高速化 | 最大絶対誤差 | 判定 |
| --- | ---: | ---: | ---: | ---: | --- |
| 3倍ダウンサンプリング | 2,551.521 ms | 43.476 ms | 58.69x | 9.4368957093138306e-16 | 一致 |
| 3倍アップサンプリング | 2,592.435 ms | 51.933 ms | 49.92x | 1.3600670589752895e-10 | 一致 |

両ケースともビット単位では差がありますが、最大絶対誤差は判定許容値 `1e-9` を下回っています。実行時間はマシンの状態によって変動します。

## 実行方法

.NET 10 SDKが必要です。プロジェクトルートで次を実行します。

```powershell
dotnet run --project Runner -c Release
```

Runnerは旧版・新版を順に実行し、実行時間、サンプル数、ビット単位で異なるサンプル数、最大絶対誤差、判定結果を表示します。比較用のバイナリデータは `artifacts` ディレクトリへ生成され、Gitの管理対象には含まれません。
